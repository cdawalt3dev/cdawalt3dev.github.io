Graph Coloring

For a theory of computation class, I was assigned to a group project to demonstrate the NP-completeness of the graph coloring problem. The problem is said to have come from early cartographers who needed to know the minimum number of colors necessary to color countries such that no two adjacent countries have the same color. This can also be modeled with a graph where nodes represent countries and connections come from shared borders.

Two algorithms were made to find a way of coloring a given graph such that no two connected nodes match in color: a deterministic and a nondeterministic one. The deterministic algorithm is brute force; it tries every combination of node colorings possible and is guaranteed to find a solution if one exists. Running out of attempts without finding a solution proves none exists. However it is unwieldy for larger graph sizes as the number of node coloring combinations becomes too time consuming to realistically process. The nondeterministic algorithm starts by coloring a node and successively colors neighboring nodes to comply with all connected nodes. A legal graph coloring is produced by coloring an additional node and then used as a basis for coloring an additional node connected to the frontier of uncolored nodes. Backtracking (uncoloring nodes) to color nodes differently if a legal graph coloring cannot be found at some step is possible. Although this algorithm will both succeed and admit failure at far fewer computations than the deterministic algorithm, it is difficult to guarantee that failure is proof that no such solution exists for a given maximum number of colors.

I created a Java program to let the user build graphs for testing both algorithms. It allows nodes to be placed, moved, deleted, and connected with other nodes through GUI. Graphs may also be saved and loaded for returning to between separate runs of the program. The user may choose a deterministic or nondeterministic algorithm to attempt to color all the nodes with a maximum number of colors specified. The default, the most connections of any one node with other nodes plus 1, is expected to be possible for any graph. A lower maximum number of colors may be possible for some graphs.

The window shows how long the algorithm has been running and the number of steps done while running. It will either end on a valid fully colored graph or admit failure. Nondeterministic algorithms may be run in step mode to show the partial graph coloring at each step. The algorithm advances when the user presses Next.
