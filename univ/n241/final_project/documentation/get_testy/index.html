Get Testy

Introduction / Process

The site was tested using the default tests of powermapper.com for verifying 508 Accessibility compliance, and webpagetest.org to test the speed. The w3 validator was used to test the page's HTML and CSS as well.



Validation

The W3 HTML Validator reported only one issue for the home page, index.html. The page option in the form, a series of checkboxes, had a label element with the "for" attribute set to "pages". "pages" was the name of the option's field at form submission and was not to the ID of any checkbox in particular.


The W3 CSS Validator did not display any issues for the site-wide CSS file, css/style.css.

Flexibility

The page resizes gracefully to fit a variety of screen sizes. Given enough width, the site centers itself and maintains a fixed width to fit as many category images as it can, side-by-side. With less width, content shrinks to fit up to a minimum of around 400px before requiring the user to scroll.

Accessibility

The test at powermapper.com revealed several minor issues.

The blank Previous and Next button href attributes on the first and last pages, Pre-Mechanical Calculators and Modern Calculators, were marked as broken links. This is despite the CSS that makes these buttons unclickable.

The test advised at least marking one radio button as selected at start and including an explicit "None" option if it should be a default or provided at all.

The test also caught a few cases of using "px" units for font size instead of something more relative, such as "em" units throughout the CSS.

The test also warned that some aesthetics and CSS features, such as rounded corners and pseudo selectors, could not be seen or work on old browsers, such as Internet Explorer 8.

The test also warned against using CSS rules that only defined color or background-color changes. However, any such changes, such as changing the table row colors, are nonessential to usability.

The test suggested giving img elements width and height attributes to restate their native size so that the page could take this into account before reading the image files. This would prevent content jumps while loading.

The test suggested using a separate color for visited links.

The test noted that the form's "textarea" element did not have a corresponding "label" element. The text meant to label the text area, "Message", should have been a label element linked to the textarea instead of a "p" element.

The test suggested using radio buttons instead of a select element if fewer than 7 choices were available.

The test suggested using a meta description tag to help describe the web page for search engines.



Browser Independence

No major differences in behavior were observed among the Microsoft Edge, Mozilla Firefox, or Google Chrome browsers. However, there were slight variations in font size and spacing among the browsers.

The custom font text was most condensed in Chrome. The text was least condensed in Edge.

Internet explorer had the most padding in the calculator categories. Firefox had slightly less padding. There was noticably even less padding in Chrome.

Chrome rendered the custom font text the most thick. Firefox drew the font the thinnest. However, Firefox rendered the font with more line height.

Resizing behavior is the same among all three browsers.



Conclusion / Corrections

The erroneous label was changed to a standard "p" element without a "for" attribte.

Several corrections were made regarding accessibility, such as using "em" units for font-size, removing blank href attributes, and changing the textarea's labeling "p" element into a "label" element.

The only corrections I did not make were outside the scope of this class or clearly did not impact the usability of the site, given other factors.
