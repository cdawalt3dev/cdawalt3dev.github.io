Shortest Path

For my data structures class, I received an assignment to use a greedy algorithm to find the ideal path in an elevation map between given start and end points. The map is a greyscale image where each pixel's color reflects a level of elevation. Although any two adjacent pixels may be traversed from one another, the greater the change in color between two pixels, the more costly the traversal would be. A straight-line path between two points is not the lowest-cost route if it takes higher cost traversals than necessary, which is often the case.

I created a program to implement the algorithm in C++. It runs from a console app that generates output images. One shows the optimum path as a path of pixels to be overlaid on the map image while the rest represent the status of pixels seen by the algorithm at different steps of the process in different ways.

To better relate to the algorithm, each pixel in the image can be represented by a node with connections to adjacent pixels/nodes. The difference in greyscale value between any adjacent pixels determines the cost a traversal between them would take. The algorithm has two sets in mind: the set of traversed nodes and the set of frontier nodes. The algorithm begins by including the start node in the traversed set and putting its adjacent nodes in the frontier set. Each traversed node records the total cost of a route from the start node (with the start node, the first 'traversed' node, having a cost of 0 to be reached). The frontier node that, reached from an adjacent traversed node, would have the lowest-cost path from the start node of all fronteir nodes, is added to the traversed set. This node receives the cost of the cheapest route to the start node (the cost stored in the node that would lead to the new node plus the cost of the traversal between the nodes), and the direction to reach the previous node in the route. The new traversed node's adjacent nodes not already in either sets are added to the fronteir set as possible choices for the next traversal choice. No node's stored cost may be set twice since its first traversal from any adjacent node produced a cheaper total route from the start node than any other fronteir nodes would have at the time. Finally, once the goal node has been added to the traversed set, the goal node backtracks through the stored node directions to previous nodes in the route from the start. Printing this route in reverse gives the best route from the start node to the goal node.

The algorithm does not guarantee that each traversal choice is a step closer to the goal node. It allows the shortest path from the start nodes to each node increasingly further away to be determined by backtracking through nodes toward the start until the goal node happens to be the cheapest fronteir node to be traversed. Then, it too may be backtracked to the start as any other node picked to be a destination that is also covered in the traversed set.
