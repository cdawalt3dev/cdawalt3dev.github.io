Shortest Path

For my data structures class, I was assigned to use a greedy algorithm to find the ideal path in an elevation map from a given start point to an end point. The map is a given greyscale image where each pixel reflects a level of elevation; any pixel can travel to any adjacent pixel. The greater the change in greyscale value (elevation) between two pixels, the more costly the traversal between them would be. A straight-line route is not the lowest-cost route if it takes higher-cost traversals than necessary, which is often the case.

I created a program to implement the algorithm in C++. It runs from a console app that generates output images. One shows the optimum path as a path of pixels on a blank image while the rest represent the status of pixels seen by the algorithm at different increments in different ways.

To better relate the pixels to the algorithm, it is appropriate to think of each pixel in the image as a node and pixels touching in cardinal directions as having connections with each other. The algorithm has two sets in mind: the set of traversed nodes (with other information) and the set of frontier nodes. The algorithm begins by including the start node in the traversed set and putting its connected (neighboring) nodes in the frontier set. The frontier node with the lowest traversal cost is chosen to be added to the traversed set, which records the direction taken to reach it in a way that gives the lowest cost and the total cost a route of all traversals from the start to itself. Since the total cost of the total route to the node is cumulative, it is sufficient to store the cost stored in the previous node (0 for the start node) plus the cost of the best traversal that lead to this node.

The process is repeated with each changed set of frontier and traversed nodes until the node added to the traversed set happens to be the goal node (a coordinate set on the image). The directions taken to reach the current node may be taken in reverse order starting from the goal to lead back to the start node, and traveling through this series of nodes in reverse gives the lowest-cost path from the start node to the goal node. In the final output image, it resembles a path roughly carved out of a valley of similarly-colored pixels that move around mountains of greater color gradients.
