Handwriting Recognition

For my artificial intelligence class, I helped create a handwriting recognition program that attempts to identify a single digit number as part of a group project. The other group members planned out key details about the program and decided how the program should standardize drawings between trials. I implemented the ideas as a Java program using a neural network.

Initially, the neural network must be given drawings assigned the numbers they represent manually. Drawings should have some variation to adequately represent imperfections and different ways the number may be drawn. The extents of the drawn area are used to form the bounds of the image. Images are then downscaled to a standard size so that they do not vary significantly due to differences in their original drawings' extents, sizes, and proportions in the window. Since a single-line vertical one (or 'stick one') could produce nearly solid black images from its bounds, there is also a check for extreme aspect ratios. In this case, the bounds are extended horizontally to better resemble the bounds expected for any other number drawn of similar height in its place. Both the standardized samples and neural network training data are saved for quick re-use between closing and opening the program.

After enough samples have been assigned to each digit, the neural network is trained so that it may attempt to identify the number portrayed in a drawing outside the training set later. Finally, the trained neural network can be used to identify a new drawing without the user's assistance. Percentages of certainty are shown in a pane to the right. For instance, the network may strongly think that a drawing is of #8 (98%), or have a lot of uncertainty (70% for #5, but 26% for #3).
