Handwriting Recognition

For my artificial intelligence class, I created a handwriting recognition program that attempts to identify the single digit number drawn in a window as part of a group project. The other group members determined how the program should standardize drawings between different trials while I implemented it as a Java program using a neural network.

It uses a neural network of input nodes given values from the pixel shading (a matrix of boolean values, shaded or not). The greatest extents of each pixel toward the left, right, top, and bottom are used to form the bounds of the image. Images are then downscaled to a standard size so that all trials are consistent regardless of differences in the rendered number's size. There is also a check for extreme aspect ratios so that single-line drawn 1's do not produce a nearly solid black image due to working with a very tall, thin image.

The program requires sample trial drawings with the number specified by the user to train the neural network. After enough drawings to adequately represent variations in drawing the number, the user may ask the program to determine what number is shown. Percentages of certainty are shown in a pane to the right. For instance, the network may strongly think that a drawing is of #8 (98%), or have a lot of uncertainty (70% #5, but 26% #3).

All training trial drawings are stored after downsizing. They are organized by the number represented as told by the user. Trials may be moved to other number categories or deleted. Trials are automatically saved between program runs. All trials are used to train the neural network. The neural network itself is also saved so that training between runs of the program without a change in trials is not necessary.
